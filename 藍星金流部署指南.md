# 藍星金流部署指南

## 前置準備

### 1. 註冊藍星金流帳號

1. 前往 [藍星金流官網](https://www.newebpay.com/)
2. 註冊測試帳號（開發階段）或正式帳號（正式上線）
3. 登入後進入「商店資料設定」
4. 啟用需要使用的金流功能：
   - 信用卡付款
   - 定期定額付款（Period）
   - 幕前支付（MPG）

### 2. 獲取 API 憑證

在藍星金流後台的「商店資料設定」中，找到以下信息並複製：

- **商店代號**（MerchantID）：格式如 `MS35199...`
- **HashKey**：加密金鑰
- **HashIV**：加密向量

### 3. 設置回調 URL

在藍星金流後台的「商店資料設定」中，設置以下回調 URL：

- **Return URL**（前台返回）：`https://your-domain.com/payment-success`
- **Notify URL**（後台通知）：`https://your-backend-domain.com/api/v1/auth/newebpay/callback`

## 環境變量配置

在後端項目的 `.env` 文件中添加以下環境變量：

```env
# 藍星金流配置
NEWEBPAY_MERCHANT_ID=你的商店代號
NEWEBPAY_HASH_KEY=你的HashKey
NEWEBPAY_HASH_IV=你的HashIV
NEWEBPAY_ENV=test  # test: 測試環境, production: 正式環境
```

## 安裝依賴

藍星金流需要使用 AES 加密，確保後端已安裝 `pycryptodome` 套件：

```bash
cd backend
pip install pycryptodome
```

如果使用 `requirements.txt`，添加以下行：

```
pycryptodome>=3.19.0
```

## 數據庫設置

藍星金流需要存儲訂單信息，確保 Supabase 中有以下表：

### orders 表（訂單表）

```sql
CREATE TABLE IF NOT EXISTS orders (
  id VARCHAR(255) PRIMARY KEY,
  user_id UUID NOT NULL,
  amount INTEGER NOT NULL,
  item_desc TEXT,
  status VARCHAR(50) DEFAULT 'pending',
  payment_method VARCHAR(50),
  period_type VARCHAR(10),
  frequency INTEGER,
  exec_times INTEGER,
  created_at BIGINT,
  paid_at BIGINT,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### subscriptions 表（訂閱表）

```sql
CREATE TABLE IF NOT EXISTS subscriptions (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID NOT NULL UNIQUE,
  plan_type VARCHAR(50) NOT NULL,
  status VARCHAR(50) DEFAULT 'active',
  trial_ends_at TIMESTAMP WITH TIME ZONE,
  next_billing_date TIMESTAMP WITH TIME ZONE,
  payment_method VARCHAR(50),
  card_last_four VARCHAR(4),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 測試流程

### 1. 測試環境測試

在測試環境下，可以使用藍星金流提供的測試信用卡：

- **測試信用卡號**：4311-9522-2222-2222
- **有效期**：任意未過期日期（如 12/30）
- **CVV**：任意 3 位數字（如 123）

### 2. 測試步驟

1. 啟動後端服務：
   ```bash
   cd backend
   python -m uvicorn app.main:app --reload
   ```

2. 啟動前端服務：
   ```bash
   cd frontend
   npm run dev
   ```

3. 註冊新用戶
4. 選擇「免費試用」方案
5. 點擊「前往藍星金流綁定信用卡」
6. 在藍星金流頁面輸入測試信用卡信息
7. 完成付款後應跳轉到支付成功頁面

### 3. 檢查後端日誌

查看後端日誌，確認回調是否正確處理：

```
Error processing callback: ...
```

如果沒有錯誤，說明回調處理成功。

## 正式環境部署

### 1. 切換到正式環境

修改 `.env` 文件：

```env
NEWEBPAY_ENV=production
```

### 2. 更新回調 URL

在藍星金流後台將回調 URL 更新為正式環境的域名：

- **Return URL**：`https://your-production-domain.com/payment-success`
- **Notify URL**：`https://your-production-backend.com/api/v1/auth/newebpay/callback`

### 3. 測試正式環境

在正式環境中，使用真實信用卡進行小額測試（如 NT$1），確保整個流程正常。

## 常見問題

### Q1: 回調 URL 無法訪問

**原因**：回調 URL 必須是公開可訪問的 HTTPS 地址，localhost 無法接收回調。

**解決方案**：
- 開發階段使用 ngrok 或類似工具將本地服務暴露到公網
- 或使用正式環境進行測試

### Q2: 加密/解密失敗

**原因**：HashKey 或 HashIV 設置錯誤。

**解決方案**：
- 檢查 `.env` 文件中的 HashKey 和 HashIV 是否正確
- 確認沒有多餘的空格或換行符

### Q3: 訂單狀態未更新

**原因**：回調處理失敗或數據庫表不存在。

**解決方案**：
- 檢查後端日誌
- 確認 orders 和 subscriptions 表已創建
- 檢查數據庫權限

### Q4: 測試信用卡付款失敗

**原因**：測試信用卡信息錯誤或已過期。

**解決方案**：
- 使用正確的測試信用卡號碼
- 確保有效期未過期
- 檢查 CVV 是否正確

## API 端點說明

### 1. 創建一次性付款

**端點**：`POST /api/v1/auth/newebpay/create-payment`

**請求體**：
```json
{
  "user_id": "用戶ID",
  "amount": 999,
  "item_desc": "商品描述",
  "email": "用戶郵箱",
  "return_url": "https://your-domain.com/payment-success",
  "notify_url": "https://your-backend.com/api/v1/auth/newebpay/callback",
  "client_back_url": "https://your-domain.com/select-plan"
}
```

**響應**：
```json
{
  "success": true,
  "data": {
    "MerchantID": "商店代號",
    "TradeInfo": "加密後的交易資料",
    "TradeSha": "SHA256 加密字串",
    "Version": "1.6",
    "PayGateWay": "藍星金流支付網關 URL"
  }
}
```

### 2. 創建定期定額付款

**端點**：`POST /api/v1/auth/newebpay/create-period-payment`

**請求體**：
```json
{
  "user_id": "用戶ID",
  "amount": 999,
  "item_desc": "商品描述",
  "email": "用戶郵箱",
  "period_type": "M",
  "frequency": 1,
  "exec_times": 12,
  "return_url": "https://your-domain.com/payment-success",
  "notify_url": "https://your-backend.com/api/v1/auth/newebpay/callback"
}
```

**參數說明**：
- `period_type`：Y=年, M=月, D=日
- `frequency`：每隔多少期執行一次
- `exec_times`：總共執行次數

### 3. 處理回調

**端點**：`POST /api/v1/auth/newebpay/callback`

**請求參數**：
- `MerchantID`：商店代號
- `TradeInfo`：加密後的交易資料
- `TradeSha`：SHA256 加密字串
- `Version`：版本號

**響應**：
```json
{
  "status": "success"
}
```

## 安全建議

1. **保護 API 憑證**：不要將 HashKey 和 HashIV 提交到版本控制系統
2. **使用 HTTPS**：確保所有支付相關的請求都使用 HTTPS
3. **驗證回調**：在回調處理中驗證 TradeSha 的正確性
4. **記錄日誌**：記錄所有支付相關的操作，方便排查問題
5. **定期檢查**：定期檢查訂單狀態，確保沒有遺漏的訂單
